Devkit Forense

Um toolkit modular e extensível, desenvolvido para democratizar o ensino de forense digital através de uma CLI intuitiva.

  • 100% Python
  • Arquitetura de Plugins
  • Foco em Automação
user@terminal:~

➜ ~ forensic-cli --help

Usage: forensic-cli [OPTIONS] COMMAND...

Commands:

browser Análise de navegadores.

network Reconhecimento de rede.

system Infos do sistema.

Módulos Implementados

browser_forensics

Analisa artefatos de navegadores baseados em Chromium e Firefox (Histórico, Downloads, Cookies).

network_recon

Realiza varreduras ativas e passivas na rede (Port Scan, Discovery, Service ID).

mail_analyzer

Processa e-mails para identificar origem e analisar metadados (SPF, DKIM, IP Origem).

Funções por Módulo

browser

  • get_historyExtrai URLs visitadas e timestamps.
  • get_downloadsLista histórico de arquivos baixados.
  • get_cookiesExtrai cookies de sessão.

network

  • scan_portsVerifica portas abertas em um alvo.
  • network_mapIdentifica hosts ativos na rede local.
  • fingerprintDetecta SO e serviços.

email

  • parse_headerExtrai metadados (From, To, IP).
  • analyze_authValida registros SPF e DKIM.
  • extract_bodySepara o corpo da mensagem.

Estrutura do Código

O projeto segue o padrão Command Pattern utilizando Typer. Cada módulo é um subcomando isolado.

Pastas

devkit_forense/
├── core/
├── modules/
│   ├── browser/
│   ├── network/
│   └── system/
├── main.py
├── pyproject.toml
└── README.md
main.py 

import typer
from modules.network import network_app

app = typer.Typer(
    name="forensic-cli",
    help="Toolkit modular."
)

# Registro de Subcomandos
app.add_typer(network_app, name="network")

if __name__ == "__main__":
    app()
